# BlockchainStand
Курсовая работа

## Что из себя представляет данный проект?
Данный проект является реализацией лабораторного стенда по технологии Blockchain.
Он позволяет развернуть приватную Ethereum-сеть с выбором алгоритма консенсуса.

На данный момент можно выбрать только Clique (PoA).

## Что используется в данном стенде:

Технологии и образы, которые использованы в данном проекте:
- Docker
- Python
- Geth (go-ethereum)

В качестве Docker-образа был выбран: `ethereum/client-go:v1.13.15`. Данный образ позволяет запускать узел в сети.


## Возможности стенда:

На данный момент стенд позволяет:
- Инициализировать сеть
- Запустить узел валидатор
- Сделать узел валидатором
- Запустить обычный узел
- Соединить узлы между собой
- Очистить весь стенд
- Удалить ноду
- Показать информацию о сети
- Показать информацию об узлах в сети


## Скрипты:

### 1. InitializeNetwork.py
Данный скрипт предназначен для инициализации сети.


Что именно делает скрипт:
- Создает и запускает Docker-сеть с названием `blockchain_stand`.
- Создает аккаунт для узла-валидатора
- Создает файл `genesis.json`
- Создает Genesis-блок, то есть первый блок в сети
- Запускает узел-валидатор


Файл `genesis.json` определяет все правила и начальное состояние сети. Перед запуском узла этот файл должен быть создан.

Из-за особенности алгоритма `Clique (PoA)` для создания сети требуется минимум один валидатор. Это нужно для того, чтобы голосовать за следующих валидаторов в сети. Если в сети не будет валидаторов, то мы не сможем создавать следующие.

Этот и дальнейшие скрипты будут автоматически раздавать IP-адрес из созданной Docker-сети для созданных узлов. Это нужно, чтобы узлы находились в одной сети и могли общаться с друг другом.

Скрипт автоматически запускает валидатор на `HTTP-порту 8545` и `P2P-порту 30303`. Важно это понимать, чтобы не занимать в дальнейшем эти порты.



**Пример использования:**

`python3 scripts/InitializeNetwork.py clique <chainId> <period> <gasLimit> <balanceInWei> <passwordForValidatorAccount>`

### 2. CreateNewNode.py
Данный скрипт предназначен для создания и запуска новых узлов в сети.

Перед использованием данного скрипта нужно запустить `InitializeNetwork.py`
Это обязательно условия для использования вообще всех скриптов, которые написаны для стенда.

Важно помнить, что данный скрипт просто создает и запускает узел в сети. Он не связывает его с другими узлами.

Чтобы связать созданный узел нужно воспользоваться скриптом `ConnectNodes.py`

Что именно делает скрипт:
- Проверяет существование Docker-сети `blockchain_stand`
- Проверяет был ли создан `genesis.json`
- Проверяет существует ли узел-валидатор
- Создает аккаунт для узла
- Запускает узел в Docker-сети `blockchain_stand`

Скрипт автоматически раздает IP-адрес из Docker-сети `blockchain_stand` для созданного узла.

**Пример использования:**
`python3 scripts/CreateNewNode.py <nodeName> <http-порт> <p2p-порт> <passwordForNodeAccount>`

Перед использование обязательно нужно проверить заняты ли порты или нет.


### 3. DeleteNode.py
Данный скрипт предназначен для удаления узла из сети.

Он удаляет все данные и аккаунт узла. Перед этим он останавливает и удаляет запущенный контейнер.

**Пример использования:**
`python3 scripts/DeleteNode.py <nodeName>`

Скрипт спрашивает пароль администратора перед удалением.

### 4. ConnectNodes.py
Данный скрипт соединяет два узла.

Для соединения обязательно, чтобы узлы были созданы и находились в одной сети



### 5. ConnectAll.py
Данный скрипт является дополнением скрипта `ConnectNodes.py`.

Он позволяет соединить все узлы для создания полной топологии.

Если какие-то узлы не получилось соединить, то скрипт сообщит об этом.

**Пример использования:**
`python3 scripts/ConnectAll.py`
